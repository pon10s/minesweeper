# CLAUDE.md — AIへの指示ファイル

> このファイルは、Claude（AI）が毎セッションの最初に読む「約束ごと」です。
> 指示がブレないように、ここを常に最新に保ちます。

## プロジェクト概要

**スマホ対応・クラシック風マインスイーパー**

Windows のクラシックなマインスイーパー（Win95/98 風）を模倣した、スマホで遊べるマインスイーパーゲーム。
AI駆動開発で、フェーズを切りながら段階的に作る。

## 最重要ルール（必ず守ること）

1. **作業前に必ず `docs/PHASES.md` を読む** — 今どのフェーズ・どのタスクをやっているか確認する。
2. **作業後に必ず記録を残す** — 変更したら `CHANGELOG.md` に追記し、`docs/PHASES.md` のタスク状態を更新する。
3. **要件は `REQUIREMENTS.md` が正** — 迷ったらここを参照。勝手に仕様を増やさない。
4. **デザインは `docs/DESIGN.md` に従う** — 色・寸法・トーンを勝手に変えない。
5. **不明点は推測で進めず、ユーザー（朱音さん）に聞く** — 特に仕様の分岐は確認する。
6. **1タスク = 1まとまり** — 大きく作りすぎず、確認しながら進める。

## 技術方針

- **フレームワークなし**（バニラ HTML / CSS / JavaScript）。ビルド不要、ダブルクリックで開ける。
- **個人データは localStorage**（ベストタイム・プレイヤー名。端末内に保存、共有しない）。
- **オンラインランキングのみ外部送信あり**（Supabase）。送るのは「名前・難易度・タイム」だけ。anon public キーはクライアント公開可で、Supabase の RLS で保護。
- **依存ライブラリは原則なし**。必要になったら理由を添えて提案する。
- **ロジック（`src/game.js`）は純粋関数中心**にして、簡易ユニットテストできる形を保つ。
- **1ファイルが大きくなりすぎないように** 適度に分割する（game / ui / main / ranking）。

## やってはいけないこと

- 要件にない機能を勝手に追加する
- ランキング以外の外部サービスへのデータ送信（個人データは端末内が原則）
- マス数・地雷数を本家から勝手に変える（スマホ対応は「向き・寸法」で吸収する）
- 記録ファイル（CHANGELOG / PHASES）の更新を忘れる

## セッション開始時の手順（AI向け）

1. この `CLAUDE.md` を読む
2. `docs/PHASES.md` で現在のフェーズ・進行中タスクを確認
3. 必要なら `REQUIREMENTS.md` `docs/DESIGN.md` を参照
4. 作業する
5. `CHANGELOG.md` と `docs/PHASES.md` を更新して終わる
