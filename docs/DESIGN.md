# DESIGN.md — デザイン方針（クラシック Win95/98 風）

見た目のブレを防ぐための仕様。色・寸法・トーンはここに従う。

## トーン

- Windows 95/98 のマインスイーパーを模倣。グレーの3D立体感、デジタル風カウンター、黄色いニコちゃん。
- フォントはシステム標準のサンセリフ（例：`Tahoma, "MS PGothic", sans-serif`）。カウンターは等幅・デジタル風。

## 配色

| 用途 | 色 |
|---|---|
| 盤面・パネル地 | `#c0c0c0`（クラシックグレー） |
| 明るいベベル（左・上） | `#ffffff` |
| 暗いベベル（右・下） | `#808080` |
| さらに濃い縁 | `#404040` |
| 開封マスの地 | `#c0c0c0`（境界は1px `#808080`） |
| カウンター背景 | `#000000` |
| カウンター文字 | `#ff0000`（赤デジタル） |
| ニコちゃん地 | `#ffff00`（黄） |

## 数字の色（周囲の地雷数）— 本家準拠

| 数字 | 色 |
|---|---|
| 1 | 青 `#0000ff` |
| 2 | 緑 `#008000` |
| 3 | 赤 `#ff0000` |
| 4 | 紺 `#000080` |
| 5 | 茶 `#800000` |
| 6 | 水色 `#008080` |
| 7 | 黒 `#000000` |
| 8 | 灰 `#808080` |

## 立体（ベベル）表現

- 未開封マス・ボタン：上と左を明るく（`#ffffff`）、下と右を暗く（`#808080`）して「出っ張り」を表現。
- 押下・開封マス：明暗を反転、もしくは薄い `#808080` の1px枠で「へこみ／平面」を表現。
- パネル枠：外側に2〜3pxのベベルで額縁状に。

## アイコン（絵文字でクラシック寄せ）

- 地雷：💣（負け時に踏んだマスは赤背景 `#ff0000`）。
- 旗：🚩。
- 誤った旗（負け時、地雷でないのに旗）：💣 に × を重ねるなど（フェーズ7で調整可）。
- ニコちゃん：通常 🙂／押下中 😮／勝ち 😎／負け 😵。

## レイアウト（縦長スマホ前提）

```
┌─────────────────────────┐
│  [残り地雷]  [🙂]  [タイマー]   │  ← ヘッダー（パネル）
├─────────────────────────┤
│                                 │
│          盤  面（格子）          │  ← 画面幅に収める
│                                 │
├─────────────────────────┤
│  難易度: [初級][中級][上級]      │  ← 下部コントロール
│  モード: [⛏ 開く] [🚩 旗]        │  ← 親指が届く位置
│  ベスト: 00:00                   │
└─────────────────────────┘
```

## マスの寸法とフィット

- マスは正方形。1辺は `min(画面幅 ÷ 列数, 上限px)` で算出（CSS変数 or JS算出）。
- 盤面全体が画面幅に収まることを最優先（横スクロール・ズーム不要）。
- **上級は縦横を入れ替えて表示**：内部データは 16行×30列でも、画面上は 30行×16列（縦長）にして横幅を抑える。
  - ※転置の扱いは「表示だけ転置」か「データごと転置」か実装時に決め、ロジックの一貫性を保つ（フェーズ5）。

## 和音クリックのジェスチャ（スマホ）

- 開封済みの数字マスを **ダブルタップ** で和音クリック発動を基本とする。
- 実装時に操作感を確認し、必要なら「長押し」など代替も検討（決定はフェーズ6で確定し、ここに追記）。
