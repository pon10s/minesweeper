## 2026-06-08（改善 第2弾）

### ダブルタップで拡大が戻る不具合を修正
- 原因：`touch-action: manipulation` を body だけに付けていたが、touch-action は継承されないため、マス自体はダブルタップ拡大が有効のままだった（連打＝ダブルタップ判定で拡大がリセット）。
- 対処：`.ms-cell` `.ms-btn` `.ms-face` に直接 `touch-action: manipulation` を付与。ダブルタップ拡大だけ無効化し、ピンチズーム（拡大の保持）は維持。

### ヒントの優先順位＆定石の説明をわかりやすく
- **優先順位を修正**：基本（ふつうに分かる手：安全・地雷の両方）を**定石より先に**出す（基本安全→基本地雷→定石安全→定石地雷→推測）。
- **定石の説明を噛み砕き＋色分け**（minesweeper.online 風）：
  - オレンジ＝見比べる2つの数字／**青＝共通マス**／緑＝安全／赤＝地雷。`.hint-common` 追加。
  - 説明を段階的な文章に（各数字の地雷数→共通で決まる数→残りがどこに入るか→結論）。`white-space: pre-line` で改行表示。
  - `findHint` の定石結果に `common`・各数字の `number/need` を持たせて具体的に説明。
- 基本ルールの文も明確化（「旗が○こ立ってるから残りは安全」「開いてないマスが数字とちょうど同じだから全部地雷」）。
- テストは 44 PASS 維持。

### ヒントに定石（部分集合の法則）を追加＋レイアウト修正
- **部分集合の法則**を `findHint` に追加（②段目）。2つの数字の隠れ範囲が包含関係のとき差分マスを確定し、1-1・1-2・1-2-1・1-1-2 等の定石をカバー。根拠の数字2つをオレンジ枠、確定マスを緑/赤、理由文を表示。返す優先順位：基本安全→定石安全→基本地雷→定石地雷→推測。
- `findHint` の戻り値を `froms`（根拠数字の配列）＋`rule`('basic'|'subset') に整理。
- **レイアウト修正**：ヒントの長い説明文がパネル幅を広げて「盤面が左寄り＋右に空白」になる不具合を解消（`.ms-hint-msg` を width:0 + min-width:100% で盤面幅に折り返し）。
- **盤面の右空白を解消**：ヘッダーが盤面より幅広いと盤面の枠だけ広がりマスが左寄り＋右に空白ができていた不具合を修正（`.ms-board` を width:fit-content + margin:auto で枠をマスに密着＆中央寄せ）。初級〜上級で右空白0・横スクロール無しを確認。
- `tests/test.js`：部分集合（1-2 定石）のテスト追加。**44 PASS / 0 FAIL**。
- 進め方：要件を REQUIREMENTS に明記→製造→ローカル検証OK→公開、の順で実施。

### ヒントを「説明つきの論理ヒント」に刷新
- これまでのヒントは「地雷位置を知った上でランダムに安全マスを光らせる」だけだった（理由なし）。
- `game.js` に確定ロジック `findHint(game)` を追加（AI不要・システムで計算）。人間と同じ推論（数字＋旗）で：
  - **ルールA（安全＝緑）**：旗の数が数字に達した数字マスの、残り隠れマスは安全。
  - **ルールB（地雷＝赤）**：残り必要地雷数＝隠れマス数の数字マスの、隠れマスは地雷。
  - **正しさ保証**：旗が真の地雷である数字のみ使う（間違った旗には惑わされない）。安全を優先。
  - 確実な手が無ければ `guess`（最初の一手は運、それ以外は「推測になるよ」）。
- UI：注目する数字に**オレンジ枠**、対象マスを**緑/赤で点滅**、ヒントボタン直下に**理由文**を表示。操作で消え5秒クールダウン。ヒント使用はランキング対象外のまま。
  - その数字から確定する**隠れマスを全部**光らせる（例：『3』の周り3マス全部を赤）。文言も件数に合わせる（「赤いマス3こは全部地雷」）。`findHint` は `targets`（配列）を返す。
- `tests/test.js`：findHint の安全/地雷/手詰まり/誤旗の4ケース追加。**40 PASS / 0 FAIL**。

### 微調整

- **ピンチズームを許可**：viewport の `maximum-scale=1.0, user-scalable=no` を撤去（`width=device-width, initial-scale=1.0` のみ）。上級など小さい盤面を**拡大したまま保持**できるように（iOSの“ズームが勝手に戻る”問題を解消）。
- **タイマーを999秒で頭打ちにせず継続**：実時間は測り続け、**表示だけ3桁ループ**（999の次は000）。ベスト/ランキングは本当の秒数（mm:ss）で記録。上級の長時間プレイに対応。
  - ※ランキングDBの上限チェックを `time < 1000` → `time < 100000` に広げる必要あり（Supabaseでpolicy更新）。
